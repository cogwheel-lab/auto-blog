各項目の詳細解説
--------

### 1\. ページネーション（オフセット）対応

*   **Serper.dev / SerpApi**: Googleの仕組み（1ページ10件〜）を継承しており、パラメータでページを切り替えて2,000件以上の深掘りが可能です。
*   **Brave Search**: 最大9ページ（約180件程度）までの制限があるプランが存在するため、数千件の一括取得には不向きな場合があります。

### 2\. 料金とコストパフォーマンス

*   **Serper.dev**: 1,000リクエストあたり\$1程度と、現在のSERP（検索結果取得）APIの中では**激安の部類**です。3,000件の取得でも数ドルの予算で収まります。
*   **Google Custom Search**: 2026年末での廃止が案内されており、新規利用は推奨されません。

### 3\. 日本語検索の精度・網羅性

*   **Serper.dev / SerpApi**: Googleのエンジンを介するため、日本語の細かなニュアンス（「コーダー」「Web制作パートナー」）を最も正確に拾えます。
*   **Exa**: 「概念的な一致」を探すため、「募集ページ」そのものではなく「募集していそうな会社」を広く探すのに適しています。

### 4\. レートリミット

*   **Serper.dev**: 50〜300 QPS（秒間リクエスト）と非常に高速で、数千件の収集も数分で完了します。
*   **Brave**: プランにより1〜50 QPSの制限があります。

執事からの推奨プラン
----------

お嬢様のM2 MacBook Proで実行されるのであれば、**Python + Serper.dev** の組み合わせが最もストレスなく、かつ安価に目的を達成できるかと存じます。

*   **理由**: DuckDuckGo (ddgs) での限界（オフセット不可）を完全に解消でき、1リクエストで10〜100件の結果を取得しつつ `page` パラメータでループを回すだけで実装可能です。
*   **補足**: 2,000件を超える場合、Googleの検索仕様上、検索ワードを少しずつ変える（例：「東京 コーダー 募集」「大阪 コーダー 募集」など）ことで、より網羅的に収集いただけます。
