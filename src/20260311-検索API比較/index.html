検索API比較：Webページ大量収集におけるオフセット制約の克服

2026年3月11日 | カテゴリ: 技術調査

結論

2,000〜3,000件の日本語ページ収集における最適解は Serper.dev です。Googleの検索結果をそのまま取得でき、日本語精度・網羅性ともに最高峰。初回2,500件の無料枠と、1,000リクエストあたり$1という激安価格で、今回の用途に最適でした。

何をしたかったか

  • 目的: 日本国内のコーダー・Web制作パートナー募集ページを網羅的に収集する
  • 対象数: 推定2,000〜3,000件規模
  • 環境: Pythonを利用

分かったこと

  • Google Custom Searchの現状: 2026年1月より新規のウェブ全体検索が不可（サイト内検索に限定）。既存ユーザーも2027年1月に廃止予定
  • Serper.devの優位性: Googleの検索結果（gl=jp, hl=ja）をそのまま低価格で叩ける。今回の3,000件規模なら無料枠＋αで完結する
  • AI特化型APIの台頭: TavilyやExaは、従来のキーワードマッチングだけでなく、AIによるフィルタリングや意味ベースの検索が可能

試したこと

日本語の網羅性と「激安」なコストを条件に、ページネーション対応の代替APIを比較調査しました。

API名 オフセット指定 1,000件単価 特徴
Serper.dev page パラメータ 約$1.00 Google生データ。無料枠2,500件。
Tavily Search max_results 約$0.30 AI特化型。ノイズ除去済みの結果。
SerpApi start パラメータ 約$15.00 高機能・高精度だが、収集用としては高価。
Exa offset 約$1.00〜 意味検索（「募集ページ」という概念）で探せる。
Brave Search offset $5.00 独自インデックス。最大9ページ制限あり。

起きた問題

  • 既存ツールの限界: DuckDuckGo Search（ddgs）を使用していたが、オフセット（ページネーション）指定ができない
  • 結果の重複: 同じ検索結果が返ってしまい、3,000件規模の網羅的な収集が困難
  • 主要APIの廃止:
    • Google Custom Search: 2026年1月より新規のウェブ全体検索が不可（サイト内検索に限定）
    • Bing Web Search: 2025年8月にサービス終了済み

推奨プラン

第一選択：Serper.dev — Googleの網羅性を維持しつつ、pageループでオフセット問題を解決。コストが激安（初回2,500件無料）でPython SDKも完備。

補完：Tavily Search — Googleで拾いきれない、あるいはノイズが多い場合に「AI検索」として併用。

実装の工夫 — 単一クエリのオフセットだけでなく、「地域名 ＋ キーワード」などの掛け合わせでリクエストを分散させ、網羅性を高めるアプローチが有効です。
